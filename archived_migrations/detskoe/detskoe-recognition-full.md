# Потолок распознавания «Детское» — с учётом пути (локально)

> ⚠️ Это **не продакшен-логика**. Скрипт `scripts/analyze_detskoe.py` намеренно использует **весь путь** (имена папок, папки сезонов, имя файла) как контекст — в отличие от приложения, которое видит только имя файла из Telegram. Цель — понять, что в принципе **можно** распознать, если задействовать структуру папок.

Прогон: `python3 scripts/analyze_detskoe.py detskoe-structure.txt` → результаты в `detskoe-recognition-full.csv`.

## 5. Как это парсится (кратко)

`scripts/analyze_detskoe.py`:

- **Сезон** — ищется в сегментах пути: `S01` (`PAW.Patrol.S01`), `Season N`, `Сезон N`, либо папка-число (`Фиксики/1`).
- **Серия** — из имени файла: число (`10`), диапазон (`127-129`), тег `s1s10` (Маша), `S01E10`.
- **Шоу vs фильм** — по карте известных сериалов + наличию папки сезона; спин-оффы (`Монстры за работой`) и числовые франшизы (Шрэк и т.п.) обрабатываются отдельно.
- **TMDB** — `/search/tv` для сериалов, `/search/movie` для фильмов, `language=ru-RU`, с кэшем по запросу.

## 6. Что это значит для приложения

Чтобы приблизиться к этому потолку в реальном приложении, есть два пути:

1. **Отдельный сканер папок** (как этот скрипт, но на TS с переиспользованием `MediaMetadataService`): подкладывать контекст пути в поле `description` классификатора. Тогда `10.mp4` в `Paw Patrol/PAW.Patrol.S01/` станет `tv_show S01E10`.
2. **При отправке через бота** — добавлять подпись (`caption`) с названием/сезоном/серией; она уже идёт в классификатор и важнее имени файла.

> Тесты для `scripts/analyze_detskoe.py` не добавлял: это разовый исследовательский скрипт (по твоей просьбе — не для продакшена). Если решишь превратить идею в фичу приложения (сканер папок), тогда покрою тестами по правилам проекта.

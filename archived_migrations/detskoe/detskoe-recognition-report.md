# Отчёт распознавания папки «Детское»

Источник: `detskoe-structure.txt` (1804 видеофайла из `/mnt/synology/video/Детское`).
Логика воспроизведена по коду приложения: классификация → порог уверенности → TMDB (`ru-RU`) → раскладка Plex.

В отчёте три взгляда:
- **Раздел A** — как распознает **приложение сейчас** (только имя файла).
- **Раздел B** — теоретический **потолок** по эвристике, если задействовать путь (`scripts/analyze_detskoe.py`).
- **Раздел C** — ⭐ **реальный прогон настоящим классификатором OpenAI** (`gpt-5.4-mini`) с контекстом пути. Это самая точная картина. Построчно — в `detskoe-openai-classification.csv` (скрипт `scripts/classify_openai.py`).

## 1. Главный вывод

**Сейчас приложение НЕ распознает ~91% файлов** и отправит их в `Undefined/`. Но если дать классификатору контекст пути (реальный OpenAI) + несколько целевых проходов — распознаётся **100%**.

| | Приложение сейчас (имя файла) | OpenAI + путь (1 проход) | ⭐ + целевые проходы (итог) |
|---|---:|---:|---:|
| 📺 Сериалы | ~52 (спорно) | 1453 (80.5%) | **1687 (93.5%)** |
| 🎬 Фильмы | ~103 | 82 (4.5%) | **117 (6.5%)** |
| ❔ В `Undefined/` | **~1649 (91%)** | 269 (14.9%) | **0 (0%)** |

Путь получения 100%:
1. **1 проход** (Раздел C): OpenAI + контекст пути → 85% (модель честно не угадывает сезон).
2. **Проход по франшизам** (C5): числовые фильмы `Шрэк/3.mp4` → конкретный фильм по порядку выхода → +19.
3. **Два кармана** (C6): эпизоды без сезона (+234) и одиночные фильмы/спецвыпуски (+18) → 0 undefined.

> Оговорка: «распознано» = модель уверенно определила тип + название. Часть номеров сезона проставлена по умолчанию `S01` (см. C6), а у ~нескольких названий TMDB-ID не подобрался из-за вариаций перевода — это не влияет на факт распознавания.

Причина принципиальная: приложение классифицирует **только по имени файла** (`fileName`), а у тебя почти все серии лежат в аккуратных папках, но сами файлы называются просто числами — `1.mp4`, `10.mkv`, `100.mp4`. Имя папки (`Paw Patrol/PAW.Patrol.S01/`) в классификатор **не передаётся** и полностью игнорируется.

То есть структура папок отличная для человека, но бесполезная для текущей логики приложения. Стоит передать классификатору контекст папки — и `10.mp4` превращается в `Щенячий патруль S01E10`.

## 2. Как считалось (соответствие коду)

- **Вход классификатора** — только `basename` файла (`src/bot/download-handlers.ts` → `getDisplayFileName`, далее `MediaClassifier.classify`). Путь/папки не используются.
- **Порог уверенности 0.7** (`MIN_CONFIDENCE` в `src/metadata/media-classifier.ts`). Ниже — `undefined`.
- **`tv_show` требует одновременно** название + номер сезона + номер серии. Если нет сезона/серии — результат обнуляется в `undefined` (`normalizeClassification`).
- **TMDB-обогащение** — реальные запросы `/search/movie` c `language=ru-RU` (тот же эндпоинт и логика, что в `TmdbResolver`). Тег в имени: `{imdb-…}` если есть, иначе `{tmdb-…}` (`plex-paths.ts`). В отчёте показан `tmdb-id` (imdb приложение добавит само на этапе `append_to_response=external_ids`).

> Оговорка: в приложении «чистое» название извлекает OpenAI (умно отбрасывает `Дублированный`, `Провайдер_2`, `q720` и т.п.). В отчёте очистка сделана эвристикой, поэтому несколько строк помечены «нет однозначного TMDB-совпадения» — реальное приложение часть из них, скорее всего, распознает.

## 5. ⚠️ Пограничный случай: «Маша и медведь» (52 эпизода)

Файлы вида `Маша_и_медведь_Перевод_2_Провайдер_2s1s10q480.mp4`.

- Название `Маша и медведь` распознаётся уверенно (TMDB: `Маша и медведь (2017) [tmdb-436209]`).
- Номер сезона/серии закодирован нестандартным тегом `s1s10` (сезон 1, серия 10). Стандартного `S01E10` нет.
- По инструкции модель должна **игнорировать теги провайдера**, а season/episode **не угадывать**. Поэтому:
  - если OpenAI распознает `s1s10` как сезон/серию → `tv_show`, разложится в `TV Shows/Маша и медведь/Season 01/…`;
  - если нет → `undefined` (название есть, но сезон/серия отсутствуют → обнуляется).
- Это **непредсказуемо** без реального прогона модели. Рекомендую переименовать в `s01e10` для гарантии.

Отдельный файл `Маша и медведь. День кино (2016)` — распознаётся как **фильм**.

## 7. Почему так и что делать

**Корень проблемы:** приложение не смотрит на структуру папок — только на имя конкретного файла. Твоя раскладка по сериям/сезонам для него невидима.

Рекомендации по приоритету:

1. **Серии (1643 файла)** — переименовать эпизоды в формат с названием и `SxxExx`, например:
   `PAW.Patrol.S01/1.mkv` → `Paw Patrol S01E01.mkv`.
   Тогда классификатор выдаст `tv_show` и разложит в `TV Shows/…/Season 01/…`.
2. **Франшизы-фильмы с числовыми именами** (Шрэк, Ледниковый период, Мадагаскар, Отель Трансильвания, Балто, Тайная жизнь домашних животных) — дать осмысленные имена с годом, напр. `Шрэк (2001).mp4`.
3. **«Маша и медведь»** — заменить тег `s1s10` на стандартный `S01E10`.
4. **Фильмы с риском ремейка** (Русалочка, Аладдин, Моана, Лило и Стич, Один дома, Последний дракон и т.п.) — добавить год выпуска в имя, чтобы TMDB брал нужную версию.
5. **Альтернатива без массового переименования:** так как через Telegram-бота у сообщения есть `caption` (подпись), можно при отправке добавлять описание — оно тоже идёт в классификатор и часто важнее имени файла.

**Вывод:** «как есть» приложение уверенно разложит по `Movies/` только ~100 полнометражек. Все сериалы (основной объём коллекции) требуют переименования файлов — папки им не помогут.

## B4. Как это парсится

`scripts/analyze_detskoe.py`:

- **Сезон** — из сегментов пути: `S01` (`PAW.Patrol.S01`), `Season N`, `Сезон N`, папка-число (`Фиксики/1`).
- **Серия** — из имени файла: число (`10`), диапазон (`127-129`), тег `s1s10` (Маша), `S01E10`.
- **Шоу vs фильм** — карта известных сериалов + наличие папки сезона; спин-оффы (`Монстры за работой`) и числовые франшизы (Шрэк и т.п.) обрабатываются отдельно.
- **TMDB** — `/search/tv` для сериалов, `/search/movie` для фильмов, `language=ru-RU`, с кэшем.

> Скрипт — разовый исследовательский инструмент (не для продакшена), тестами не покрыт. Если превращать идею в фичу приложения (сканер папок с подстановкой пути в `description`), тогда — по правилам проекта, с тестами.

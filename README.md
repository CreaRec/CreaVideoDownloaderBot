# Telegram Video Downloader

Telegram Video Downloader is a private Node.js service that listens to your Telegram bot and downloads videos or document-style video files into a configured directory using GramJS.

It uses two Telegram identities:

- A bot token, created with BotFather, to receive files you send or forward to the bot.
- A GramJS user session per allowed Telegram user, created with your Telegram API ID/API hash, to download media from that user's private chat with the bot.

## Features

- Accepts video messages and document-style video files from configured Telegram user IDs only.
- Shows download progress in Telegram and updates the final status with the saved path and file size.
- Adds an inline delete button to each completed download; deleting an active download cancels it before removing the file.
- Uses OpenAI + TMDB metadata enrichment to save files into Plex-compatible `Movies`, `TV Shows`, or `Undefined` folders.
- Provides an inline file browser for downloaded files and folders, including refresh, back navigation, and confirmed deletion.
- Reports OpenAI usage from Telegram when an OpenAI admin API key is configured.
- Supports a private `/restart` command; Docker restarts the container after the process exits.

## Requirements

- Node.js 22.9.0 or newer (local development and GramJS login).
- A Telegram API ID and API hash from <https://my.telegram.org>.
- A Telegram bot token from BotFather.
- Your numeric Telegram user ID for `telegram.userSessions`. Each allowed user needs a GramJS session created with `npm run login`.
- Optional: an OpenAI API key for media classification.
- Optional: a TMDB API key for verified Plex metadata IDs (`imdb`, `tvdb`, `tmdb`).
- Optional: an OpenAI admin API key for usage reporting.
- Production: Docker on the server and GHCR access (see [docs/docker.md](docs/docker.md)).

## Configuration

Copy the example settings file and fill in your values:

```sh
cp config/settings.example.json config/settings.json
```

`config/settings.json` is ignored by git because it contains secrets. You can also point the app at another settings file with `SETTINGS_PATH`.

Settings:

- `telegram.apiId`: Telegram API ID for GramJS.
- `telegram.apiHash`: Telegram API hash for GramJS.
- `telegram.userSessions`: map of Telegram user ID to GramJS string session. Keys define who may use the bot; values are generated by `npm run login`.
- `telegram.botToken`: bot token from BotFather.
- `telegram.botUsername`: bot username without `@`.
- `download.directory`: directory where files are saved. In Docker this must be `/downloads` (host path comes from `DOWNLOAD_DIR` in `.env`).
- `download.overwriteExisting`: whether existing filenames may be overwritten.
- `openai.apiKey`: optional OpenAI API key for classifying media before saving.
- `openai.adminApiKey`: optional OpenAI admin API key for `/usage`.
- `openai.model`: OpenAI model used for media classification.
- `openai.instructionsPath`: prompt file used by the media classifier.
- `openai.usageStartDate`: optional `YYYY-MM-DD` start date for the default `/usage` report.
- `tmdb.apiKey`: optional TMDB API key for Plex ID enrichment.
- `tmdb.language`: TMDB language for titles and episode names, default `ru-RU`.
- `app.logLevel`: `debug`, `info`, `warn`, or `error`.

## Quick Start (production)

Production runs in Docker. Bootstrap the server once, then deploy only via git:

1. Follow the one-time checklist in [docs/docker.md](docs/docker.md) (GHCR login, deploy dir, settings, stop old systemd).
2. Merge to `main` — GitHub Actions builds the image, pushes to GHCR, and runs `docker compose pull/up` on the server.

For GramJS sessions, run login from a full checkout (see [docs/docker.md](docs/docker.md)):

```sh
npm install
cp config/settings.example.json config/settings.json
npm run validate:settings
npm run login -- --user-id <your_telegram_user_id>
```

Copy the updated `settings.json` to the server deploy directory and restart the container.

Send or forward a video to your bot from a user listed in `telegram.userSessions`. The service saves media under the configured download volume.

## Supported Bot Commands

- `/start`: confirms the bot is available, shows the Files reply keyboard, and explains the expected video/document workflow.
- `Files` (reply keyboard) or `/files`: opens the inline file browser for `download.directory`.
- `/usage`: shows OpenAI usage for the configured range, or month-to-date when no `openai.usageStartDate` is set.
- `/usage today`: shows OpenAI usage for the current UTC day.
- `/usage month`: shows OpenAI usage for the current UTC month.
- `/restart`: replies with confirmation, exits the app, and lets Docker restart the container.

Any other message from an allowed user receives a short help response. Messages from users outside `telegram.userSessions` are rejected or ignored.

## Download Layout

When OpenAI classification is configured, the bot enriches metadata with TMDB when `tmdb.apiKey` is set, then saves files in Plex-compatible layout:

- Films: `Movies/Inception (2010) {imdb-tt1375666}/Inception (2010) {imdb-tt1375666}.mkv`
- TV episodes: `TV Shows/Breaking Bad (2008) {tvdb-81189}/Season 03/Breaking Bad (2008) - s03e04 - Episode Title.mkv`
- Unclassified media: `Undefined/<original-or-generated-name>`

The classifier uses both the Telegram filename and message caption/description. If OpenAI or TMDB is not configured, files fall back to `Undefined` or to title/year-only Plex paths without ID tags.

If OpenAI is not configured or classification fails, files are saved under `Undefined`. When `download.overwriteExisting` is `false`, the app picks an available filename instead of replacing an existing file.
